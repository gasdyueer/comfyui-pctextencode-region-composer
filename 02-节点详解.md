# 节点详解

本文档详细介绍 ComfyUI Prompt Control 提供的所有节点，包括参数说明和使用场景。

---

## 一、调度节点（Lazy 节点）

Lazy 节点是 PC 的核心，它们利用 ComfyUI 的懒执行机制，**自动将带调度语法的提示词展开为完整的节点图**。最大的好处是：如果调度的一部分没有变化，ComfyUI 的缓存机制可以避免重新编码。

### 1. PC: Schedule prompt（`PCLazyTextEncode`）

> 日常使用最频繁的节点

**功能**：解析提示词中的调度语法，自动生成 `PCTextEncode` + `ConditioningSetTimestepRange` 节点图。

**输入**：

| 参数 | 类型 | 说明 |
|------|------|------|
| clip | CLIP | CLIP 模型 |
| text | 字符串 | 提示词（支持调度、LoRA标记、宏等全部语法） |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| conditioning | CONDITIONING | 编码后的条件 |

**使用示例**：
```
a [large::0.1] [cat|dog:0.05] [<lora:somelora:0.5:0.6>::0.5]
[in a park:in space:0.4]
```

**特殊功能 — NODE()**：在提示词中使用 `NODE(节点类名, 文本参数名)` 可以指定用其他兼容节点来编码提示词。例如：
- `NODE(CLIPTextEncode)` — 使用 ComfyUI 原始的 CLIP 编码节点（不含 PC 高级功能）
- 默认等价于 `NODE(PCTextEncode, text)`

---

### 2. PC: Schedule prompt (Advanced)（`PCLazyTextEncodeAdvanced`）

**功能**：与 `PCLazyTextEncode` 相同，但增加了多遍工作流所需的过滤和步数控制功能。

**输入**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| clip | CLIP | — | CLIP 模型 |
| text | 字符串 | "" | 提示词 |
| tags | 字符串 | "" | 标签过滤，逗号分隔。用于多遍渲染选择特定调度分支 |
| start | 浮点数 | 0.0 | 调度起始百分比 (0.0-1.0) |
| end | 浮点数 | 1.0 | 调度结束百分比 (0.0-1.0) |
| num_steps | 整数 | 0 | 总采样步数。设置后可在提示词中使用**绝对步数**而非百分比 |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| conditioning | CONDITIONING | 编码后的条件 |

**使用场景**：

1. **使用绝对步数**：设 `num_steps=20`，提示词中可写 `[cat:dog:10]` 表示第 10 步切换
2. **多遍渲染**：配合 `tags` 参数，同一提示词在不同遍次输出不同内容
3. **截取调度片段**：用 `start`/`end` 只使用调度的某一部分

---

### 3. PC: Schedule LoRAs（`PCLazyLoraLoader`）

**功能**：从提示词中读取 LoRA 表达式，自动构建 `LoraLoader` + `CreateHookLora` + `CreateHookKeyframe` 节点图，实现 LoRA 的加载与调度。

**输入**：

| 参数 | 类型 | 说明 |
|------|------|------|
| model | MODEL | 模型（可选） |
| clip | CLIP | CLIP 模型（可选） |
| text | 字符串 | 包含 LoRA 标记的提示词 |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| model | MODEL | 加载了 LoRA 的模型 |
| clip | CLIP | 加载了 LoRA 的 CLIP |

**使用示例**：
```
a cat <lora:catlora:1> [<lora:detailer:0.8>::0.5]
```
- `catlora` 全程生效（直接加载到模型上）
- `detailer` 通过 Hook 机制只在前 50% 生效

---

### 4. PC: Schedule LoRAs (Advanced)（`PCLazyLoraLoaderAdvanced`）

**功能**：与 `PCLazyLoraLoader` 相同，增加了 Hook 输出和过滤功能。

**输入**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| model | MODEL | — | 模型（可选） |
| clip | CLIP | — | CLIP 模型（可选） |
| text | 字符串 | "" | 提示词 |
| apply_hooks | 布尔 | True | 是否自动将 Hook 应用到 CLIP。设为 True 时无需手动连接 HOOKS 输出 |
| tags | 字符串 | "" | 标签过滤 |
| start | 浮点数 | 0.0 | 调度起始百分比 |
| end | 浮点数 | 1.0 | 调度结束百分比 |
| num_steps | 整数 | 0 | 总采样步数 |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| model | MODEL | 加载了 LoRA 的模型 |
| clip | CLIP | 加载了 LoRA 的 CLIP |
| hooks | HOOKS | 生成的 Hook 组。如果 `apply_hooks=True`，Hook 已自动应用到 clip，此输出供你另作他用 |

---

## 二、编码节点

### 5. PC: Text Encode (no scheduling)（`PCTextEncode`）

**功能**：对**单段提示词**进行高级编码。支持 AND、BREAK、STYLE、TE、MASK 等全部高级语法，但**不支持**调度语法 `[a:b:0.5]` 和 LoRA 标记。

> 通常你不需要直接使用此节点，它由 Lazy 节点自动调用。

**输入**：

| 参数 | 类型 | 说明 |
|------|------|------|
| clip | CLIP | CLIP 模型 |
| text | 字符串 | 提示词（不支持调度语法） |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| conditioning | CONDITIONING | 编码后的条件 |

**注意**：`PCTextEncode` 不会忽略 `<lora:...:1>`，会将其当作普通文本处理。如果需要 LoRA 调度，请使用 Lazy 节点。

---

### 6. PC: Text Encode with Range (no scheduling)（`PCTextEncodeWithRange`）

**功能**：与 `PCTextEncode` 相同，但可指定时间步范围。在配合已调度的 LoRA 时可能更高效。

**输入**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| clip | CLIP | — | CLIP 模型 |
| text | 字符串 | — | 提示词 |
| start | 浮点数 | 0.0 | 起始百分比 |
| end | 浮点数 | 1.0 | 结束百分比 |

**输出**：同 `PCTextEncode`

---

## 三、工具节点

### 7. PC: Attach Mask（`PCAddMaskToCLIP`）

**功能**：将遮罩附加到 CLIP 对象上，以便在提示词中通过 `IMASK(index)` 引用。

**输入**：

| 参数 | 类型 | 说明 |
|------|------|------|
| clip | CLIP | CLIP 模型 |
| mask | MASK | 遮罩（可选） |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| clip | CLIP | 附加了遮罩信息的 CLIP |

**用法**：多次使用此节点会**追加**遮罩而非覆盖。在提示词中用 `IMASK(0)` 引用第一个附加的遮罩，`IMASK(1)` 引用第二个，依此类推。

---

### 8. PC: Attach Mask (multi)（`PCAddMaskToCLIPMany`）

**功能**：`PCAddMaskToCLIP` 的多输入版本，一次可附加最多 4 个遮罩。

**输入**：

| 参数 | 类型 | 说明 |
|------|------|------|
| clip | CLIP | CLIP 模型 |
| mask1-mask4 | MASK | 4 个可选遮罩输入 |

**输出**：同 `PCAddMaskToCLIP`

**提示**：空输入会被跳过，所以如果你只在 `mask3` 接了遮罩，它在提示词中的索引仍然是 0（因为前面的空输入不计入）。

---

### 9. PC: Configure PCTextEncode（`PCSetPCTextEncodeSettings`）

**功能**：配置 `PCTextEncode` 的默认参数值，将配置信息附加到 CLIP 对象上。

**输入**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| clip | CLIP | — | CLIP 模型 |
| mask_width | 整数 | 512 | 默认遮罩宽度（像素） |
| mask_height | 整数 | 512 | 默认遮罩高度（像素） |
| sdxl_width | 整数 | 1024 | SDXL 默认宽度 |
| sdxl_height | 整数 | 1024 | SDXL 默认高度 |
| sdxl_target_w | 整数 | 1024 | SDXL 目标宽度 |
| sdxl_target_h | 整数 | 1024 | SDXL 目标高度 |
| sdxl_crop_w | 整数 | 0 | SDXL 裁剪宽度 |
| sdxl_crop_h | 整数 | 0 | SDXL 裁剪高度 |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| clip | CLIP | 附加了配置的 CLIP |

**用法**：将此节点放在 CLIP 通路中、`PCTextEncode` 之前。之后在提示词中使用 `MASK()`、`SDXL()` 时，默认值会被此节点的设置覆盖。

---

### 10. PC: Extract Scheduled Prompt（`PCExtractScheduledPrompt`）

**功能**：解析调度提示词，返回在指定时间点的纯文本提示词。不执行编码，只做文本解析。

**输入**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| text | 字符串 | — | 调度提示词 |
| at | 浮点数 | 1.0 | 时间点 (0.0-1.0) |
| tags | 字符串 | "" | 标签过滤 |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| 字符串 | STRING | 在该时间点的提示词文本 |

**用法**：可用于调试，或在不编码的情况下提取某步的提示词传给其他节点。

---

### 11. PC: Expand Macros（`PCMacroExpand`）

**功能**：展开提示词中的 `DEF` 宏定义，返回展开后的文本。

**输入**：

| 参数 | 类型 | 说明 |
|------|------|------|
| text | 字符串 | 含宏定义的提示词 |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| 字符串 | STRING | 展开宏后的文本 |

**注意**：`PCLazyTextEncode` 和 `PCLazyLoraLoader` 会自动展开宏，所以通常不需要手动使用此节点。但如果你使用 `PCTextEncode`（非调度版本），则需要先用此节点展开宏。

---

### 12. PC: Configure Logging (for debug)（`PCSetLogLevel`）

**功能**：设置 Prompt Control 的日志级别，用于调试。

**输入**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| clip | CLIP | — | CLIP 模型（直通） |
| level | 选项 | INFO | 日志级别：INFO / DEBUG / WARNING / ERROR |

**输出**：CLIP（不变）

**提示**：也可通过环境变量 `PROMPTCONTROL_DEBUG=1` 启用调试日志。

---

## 四、实验性节点

> ⚠️ 实验性节点的语法和行为可能在将来版本中变更。

### 13. PC: LoRA Hooks From Text (non-lazy)（`PCLoraHooksFromText`）

**功能**：从调度提示词中创建 LoRA Hook 组。这是非 Lazy 版本，直接在节点内创建 Hook 而非生成节点图。

**输入**：

| 参数 | 类型 | 说明 |
|------|------|------|
| text | 字符串 | 含 LoRA 标记的提示词 |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| hooks | HOOKS | LoRA Hook 组 |

---

### 14. PC: Attention Couple (batch negative)（`PCAttentionCoupleBatchNegative`）

**功能**：将正面条件的注意力耦合 Hook 批量应用到负面条件上，使结果与 pamparamm 的原始实现一致，同时提升性能。

**输入**：

| 参数 | 类型 | 说明 |
|------|------|------|
| positive | CONDITIONING | 正面条件（含注意力耦合 Hook） |
| negative | CONDITIONING | 负面条件 |

**输出**：

| 参数 | 类型 | 说明 |
|------|------|------|
| positive | CONDITIONING | 正面条件（不变） |
| negative | CONDITIONING | 批量应用了注意力耦合的负面条件 |

**用法**：在使用 `COUPLE` 语法时，将正面和负面条件都接入此节点可获得最佳效果。

---

## 节点选择速查

| 我想... | 使用节点 |
|---------|---------|
| 在提示词中切换内容 | `PC: Schedule prompt` |
| 在提示词中调度 LoRA | `PC: Schedule LoRAs` |
| 同时调度提示词和 LoRA | 两个节点搭配使用（共享同一提示词） |
| 只用高级编码，不要调度 | `PC: Text Encode (no scheduling)` |
| 在提示词中使用自定义遮罩 | `PC: Attach Mask` + `IMASK()` |
| 配置 SDXL 参数默认值 | `PC: Configure PCTextEncode` |
| 多遍渲染，每遍选不同内容 | `PC: Schedule prompt (Advanced)` + tags |
| 使用绝对步数而非百分比 | `PC: Schedule prompt (Advanced)` + num_steps |
| 调试提示词解析 | `PC: Extract Scheduled Prompt` / `PC: Configure Logging` |
