## 1. 调度语法

调度语法让你在采样过程中动态切换提示词内容，类似 A1111 的提示词编辑功能。

### 基本切换 `[before:after:X]`

在采样进度 `X`（0.0 到 1.0）处，从 `before` 切换到 `after`。

```
a [red:blue:0.5] cat
```
- 0%~50%：a red cat
- 50%~100%：a blue cat

**`after` 可以留空**：
```
[clouds::0.7]  → 70%前是 clouds，70%后为空
```

**`before` 可以省略**：
```
[clouds:0.3]  → 等同于 [:clouds:0.3]，30%前为空，30%后是 clouds
```

### 嵌套调度

调度可以嵌套，实现多阶段切换：

```
a [red:[blue::0.7]:0.5] cat
```
- 0%~50%：a red cat
- 50%~70%：a blue cat
- 70%~100%：a cat

### 范围表达式 `[before:during:after:start,end]`

三段式范围表达，更直观地指定多个阶段：

```
prompt [morning:noon:night:0.2,0.7]
```
- 0%~20%：prompt morning
- 20%~70%：prompt noon
- 70%~100%：prompt night

**简写**：
- `[noon:0.2,0.7]` → 等同于 `[:noon::0.2,0.7]`
- `[noon:night:0.2,0.7]` → 等同于 `[:noon:night:0.2,0.7]`
- `[morning:noon:0.2]` → 等同于 `[morning:noon:0.2,1.0]` → 等同于 `[morning:noon:0.2]`

### 使用绝对步数

在 **Advanced** 节点中设置 `num_steps` 后，可以在提示词中使用步数代替百分比：
- 含小数点的数字 → 百分比（如 `0.5` = 50%）
- 不含小数点的整数 → 绝对步数（如 `10` = 第 10 步）

```
# num_steps = 20
[cat:dog:10]  → 第10步切换（等同于 0.5）
```

## 2. LoRA 语法

在 Lazy 节点中，LoRA 可以通过 `<lora:...>` 标记来加载和调度。

### 基本 LoRA 加载

```
<lora:名称:权重>
<lora:名称:模型权重:CLIP权重>
```

```
<lora:cat_style:0.8>          → 模型和CLIP权重都是 0.8
<lora:cat_style:0.8:0.5>     → 模型权重 0.8，CLIP权重 0.5
```

### LoRA 调度

将 LoRA 放入调度表达式中即可实现调度：

```
[<lora:detail:0.8>::0.5]      → detail LoRA 在前 50% 生效
[<lora:style1:1>:<lora:style2:1>:0.5]  → 50%前用 style1，50%后用 style2
```

**全程生效的 LoRA** 直接写在提示词中（不放在调度内），会被直接加载到模型上，更高效：

```
<lora:base_style:1> [<lora:extra_detail:0.5>::0.3]  → base_style全程，extra_detail仅前30%
```

### LoRA 名称查找规则

PC 按以下顺序查找 LoRA 文件：

1. **精确匹配**：文件名完全匹配（不含扩展名）
2. **子目录搜索**：在所有 LoRA 目录的子目录中搜索同名文件
3. **空格转下划线**：将空格替换为下划线后重新搜索（兼容自动补全脚本）
4. **关键词模糊匹配**：按空格拆分搜索词，查找包含所有关键词的文件。仅当结果唯一时才匹配

```
<lora:cats:1>                → 匹配 cats.safetensors 或 sd15/animals/cats.safetensors
<lora:XL/sdxllora:0.5>      → 指定子目录路径
<lora:cats and dogs:1>      → 自动尝试 cats_and_dogs.safetensors
<lora:cats xl:1>            → 模糊匹配含 "cats" 和 "xl" 的唯一文件
```

也可以使用完整路径（含扩展名），与 ComfyUI 原生 `LoRALoader` 一致。

## 3. 交替与序列

### 交替 `[a|b:X]`

每隔 `X` 比例交替切换提示词，默认 `X=0.1`：

```
[red|blue:0.1]   → 每10%交替：红→蓝→红→蓝...
[cat|dog|bird:0.05]  → 三选项交替，每5%切换
```

### 序列 `[SEQ:a:N1:b:N2:c:N3]`

按顺序依次切换的快捷写法，相当于嵌套的调度表达式：

```
[SEQ:sunrise:0.2:noon:0.5:sunset:0.8]
```
等价于：
```
[sunrise:[noon:[sunset::0.8]:0.5]:0.2]
```
- 0%~20%：sunrise
- 20%~50%：noon
- 50%~80%：sunset
- 80%~100%：空

## 4. 提示词组合

### AND — 组合提示词

用 `AND` 分隔的提示词会被独立编码后用 `ConditioningCombine` 组合：

```
cat :1 AND dog :2
```

**权重**：`AND` 后的数字是该段的权重（默认 1）。权重为 0 的段会被**完全跳过**：

```
cat [\:0::0.5] AND dog   → 前50%只有cat（\:0 是权重0，被跳过），后50%同时有cat和dog
```

> 注意：`:` 需要转义，否则会被当作调度语法。

**引号内的 AND 不触发**：`Text saying "CAT AND DOG"` → AND 被当作普通文本。

### BREAK — 分块编码

`BREAK` 将提示词分成独立的 token 块，每块填充到编码器最大长度后分别编码：

```
a cat BREAK sitting on a chair BREAK in a room
```

> 对于某些编码器（如 t5），BREAK 没有意义，会被自动忽略。

### CAT — 拼接编码

`CAT` 分别编码每段提示词，然后将张量拼接为单个条件：

```
a cat CAT a dog   → 等同于 ComfyUI 的 ConditioningConcat
```

### AVG() — 加权平均

```
prompt1 AVG(0.3) prompt2
```

分别编码两段提示词，用 `ConditioningAverage` 按 0.3 的权重混合。默认权重 0.5。

可链式使用：
```
p1 AVG() p2 AVG() p3   → 先混合 p1 和 p2，再将结果与 p3 混合
```

## 5. 标签过滤

使用 **Advanced** 节点的 `tags` 参数，可以基于标签选择调度的分支。标签必须是大写字母和下划线。

```
提示词：
a [black:blue:X] [cat:dog:Y] [walking:running:Z] in space

tags 设为 "x,z" 时 → a blue cat running in space
tags 设为 "y" 时   → a black dog walking in space
```

三段式 `[a:b:c:TAG]` 中的 `b` 会被忽略，等同于 `[a:c:TAG]`。

**典型用途**：多遍渲染，第一遍用一组内容，第二遍用另一组。

## 6. 权重风格

### STYLE() — 设置权重解析方式

```
STYLE(weight_interpretation, normalization)
```

**权重解析方式**：

| 名称 | 说明 |
|------|------|
| comfy | ComfyUI 默认方式（也是 PC 默认） |
| comfy++ | ComfyUI 增强版 |
| A1111 | Auto1111 风格 |
| compel | Compel 风格 |
| down_weight | 降权方式 |
| perp | 垂直投影方式 |

**归一化方式**（可用 `+` 组合）：

| 名称 | 说明 |
|------|------|
| none | 不归一化（默认） |
| length | 按长度归一化 |
| mean | 按均值归一化 |

```
STYLE(A1111)                    → 使用 A1111 权重解析
STYLE(A1111, length+mean)      → A1111 + 长度和均值归一化
STYLE(comfy, mean+length)      → ComfyUI + 均值和长度归一化
```

**继承规则**：第一个 `STYLE` 会作为后续 AND 段的默认值，除非被覆盖：

```
STYLE(A1111) prompt1 AND prompt2 AND prompt3
→ 三段都用 A1111

prompt1 AND STYLE(A1111) prompt2 AND prompt3
→ prompt1 用默认 comfy，prompt2 用 A1111，prompt3 用默认 comfy
```

## 7. 多编码器提示

### TE() — 按编码器指定提示词

对于使用多个文本编码器的模型（如 SDXL 有 CLIP L 和 CLIP G，Flux 有 T5 XXL）：

```
TE(l=a cute cat) TE(g = (a beautiful dog:1.1)) TE(t5xxl=a tiger)
```

- `l` → CLIP L
- `g` → CLIP G
- `t5xxl` → T5 XXL（Flux 文本编码器）

**使用 `TE(help)` 可在日志中查看当前模型可用的编码器键名。**

**注意事项**：
- 设置了 `TE` 的编码器会**覆盖**该编码器对应的外部提示词
- 多个 `TE()` 调用会以空格连接：`TE(l=foo)TE(l=bar)` 等同于 `TE(l=foo bar)`
- `AND` 和 `BREAK` 在 `TE()` 内不起作用，但 `SHIFT`、`SHUFFLE` 和 `OLDBREAK` 可以使用

### SDXL() — 设置 SDXL 参数

```
SDXL(1024 1024, 1024 1024, 0 0)
```

参数：`SDXL(宽 高, 目标宽 目标高, 裁剪宽 裁剪高)`

默认值 `SDXL(1024 1024, 1024 1024, 0 0)`，可通过 `PC: Configure PCTextEncode` 节点修改默认值。

### TE_WEIGHT() — 设置编码器权重

```
TE_WEIGHT(g=0.25, l=0.75)
```

对指定编码器的输出乘以权重。也支持 `l_pooled` 等 pooled 输出权重。

```
TE_WEIGHT(all=0.5)   → 所有编码器权重设为 0.5
```

## 8. Cutoff 隔离

Cutoff 可以隔离提示词中不同概念的相互影响，减少"颜色串色"等问题。

### 语法

```
[区域文本:目标词:权重:strict_mask:start_from_masked:padding_token]
```

### 基本示例

```
a group of animals, [CUT:white cat:white], [CUT:brown dog:brown]
```

读取方式：基础提示是 "a group of animals, white cat, brown dog"，但 CUT 会让 "white" 对 "cat" 的影响被隔离，"brown" 对 "dog" 的影响被隔离，从而减少"棕色猫"或"白色狗"的情况。

### 目标词匹配

- 空格分隔的词会被**分别**遮罩：`[CUT:green apple, red apple:green apple]` → 遮罩 "green" 和 "apple" 两个词
- 用下划线连接的词会被**作为整体**遮罩：`[CUT:green apple, red apple:green_apple]` → 只遮罩 "green apple" 整体
- 转义下划线：`\_` 表示字面下划线

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 区域文本 | — | 要隔离的区域提示词 |
| 目标词 | — | 要遮罩的关键词 |
| weight | 1.0 | 区域权重 |
| strict_mask | 1.0 | 严格遮罩强度 |
| start_from_masked | 1.0 | 从遮罩开始的比例 |
| padding_token | + | 用于替换被遮罩 token 的填充符号 |

**参数继承**：`strict_mask`、`start_from_masked` 和 `padding_token` 在多个 CUT 中，**最后一个设置的值会成为后续 CUT 的默认值**。

> ⚠️ Cutoff 不适用于非 CLIP 编码器的模型（如 Flux），且语法可能在将来变更。

## 9. 宏定义

### DEF — 定义和使用宏

宏在所有其他解析之前展开，可以嵌套但不可递归：

```
DEF(MYCAT=a cute orange cat)
I have a MYCAT sitting on a MYCAT
→ I have a cute orange cat sitting on a cute orange cat
```

宏与调度结合：
```
DEF(MYMACRO=this is a prompt)
[(MYMACRO:0.6):(MYMACRO:1.1):0.5]
→ [(this is a prompt:0.6):(this is a prompt:1.1):0.5]
```

### 带参数的宏

```
DEF(MYMACRO=[(prompt $1:$2):(prompt $1:$3):$4])
MYMACRO(test; 1.1; 0.7; 0.2)
→ [(prompt test:1.1):(prompt test:0.7):0.2]
```

参数用**分号**分隔。

### 带默认值的宏

```
DEF(MACRO(example; 0; 1)=[$1:$2,$3])
MACRO              → [example:0,1]
MACRO(test; 0.2)   → [test:0.2,1]
```

### 未指定参数的行为

```
DEF(mything=a "$1" b "$2")
mything       → a "$1" b "$2"    （未调用，保持原样）
mything()     → a "" b "$2"      （调用但无参，$1变空，$2不变）
mything(A)    → a "A" b "$2"     （只给$1，$2不变）
```

> **注意**：`PCTextEncode` 不会自动展开宏，需用 `PCMacroExpand` 节点或 Lazy 节点。

## 10. 其他函数

### SHUFFLE() — 随机排列

```
SHUFFLE(seed, separator, joiner)
```

按 `separator`（默认 `,`）拆分提示词，随机排列后用 `joiner`（默认 `,`）连接：

```
SHUFFLE() cat, dog, tiger, mouse   → cat, dog, mouse, tiger（结果随机）
SHUFFLE(42) cat, dog, tiger        → 使用固定种子42
```

### SHIFT() — 循环移位

```
SHIFT(steps, separator, joiner)
```

将元素左移 `steps` 位：

```
SHIFT(1) cat, dog, tiger, mouse   → dog, tiger, mouse, cat
SHIFT(1,;) cat, dog ; tiger, mouse → tiger, mouse, cat, dog
SHIFT(1,, ) cat,dog               → dog cat（用空格连接）
```

可链式使用：`SHUFFLE() SHIFT(1) cat, dog, tiger, mouse`

> 注意：括号内的分隔符会被忽略，以避免破坏强调语法 `(word:1.5)`。

### NOISE() — 添加噪声

```
NOISE(weight, seed)
```

向条件张量添加随机噪声。`weight` 范围 0~1，`seed` 可选。

```
NOISE(0.1, 42) a cat   → 添加 10% 噪声，种子 42
```

### Embedding 替代语法

`<emb:xyz>` 是 `embedding:xyz` 的替代写法，用于避免与调度语法冲突：

```
<emb:my_embedding>   → 等同于 embedding:my_embedding
```

因为 `[embedding:xyz:0.5]` 会被误解析为调度表达式（从 "embedding" 切换到 "xyz"）。

## 11. 注释与转义

### 注释

`#` 后的内容到行末被当作注释，会被移除：

```
a cat # 这是注释
→ a cat
```

### 转义

| 原字符 | 转义写法 | 说明 |
|--------|----------|------|
| `#` | `\#` | 避免被当作注释 |
| `:` | `\:` | 避免被当作调度语法 |
| `\` | `\\` | 字面反斜杠 |
| `,` | `\,` | 函数参数中的逗号 |

转义仅在会与其他语法冲突时才需要。例如 `\o/` 中的 `\` 不需要转义，但 `[embedding:a:0.5]` 中的 `:` 需要转义为 `[embedding\:a\:0.5]`。

### 引号保护

双引号内的 `AND`、函数调用等不会被解析：

```
Text saying "CAT AND DOG"   → AND 被当作普通文本
```

## 12. 处理顺序

了解处理顺序有助于理解各语法之间的交互：

1. **DEF 宏展开** — 所有宏被替换
2. **调度展开** — `[a:b:X]` 被解析，对每个调度段：
   - 按 **AND** 分割，对每段：
     - 按 **COUPLE** 分割
     - 评估 MASK、Cutoff 等函数
     - 按 **AVG()** 或 **CAT** 分割
       - 评估 **TE()** 设置各编码器提示词
       - 评估 **BREAK**
       - 其余处理
     - 用 ConditioningAverage（AVG）或 ConditioningConcat（CAT）组合
   - 如有耦合提示，设置注意力耦合
   - 用 ConditioningCombine 组合 AND 段
3. 每个调度段用 ConditioningSetTimestepRange 限制到有效范围

**关键含义**：
- `BREAK` 可以放在 `TE()` 内部，但 `AND` 和 `CAT` 不行
- `STYLE` 在 AND 层面生效，会影响整段
- 宏在所有解析之前展开，所以宏的内容可以包含任何语法
